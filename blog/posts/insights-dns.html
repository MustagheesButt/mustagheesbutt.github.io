What is DNS?

DNS stands for Domain Name System, not Domain Name Server. It's a protocol or a set of methods and actions which defines everything happening from the requesting of a webpage by a client, through the conversion of domain name into an IP address and the processes taking place in server, to the providing or redirecting of client to the requested webpage. That was just the summary of DNS.

Note that webpages and other files are not transferred by DNS, in fact only their IP addresses are transferred.

Steps involved in DNS

  1. The Domain or URL

    2. First the client or user makes a request for a webpage in a web browser. The syntax of URL or domain which is entered in the address bar of the web browser is something like this:

    A domain name consists of several parts called labels. They are separated by dots, such as "www.rustedbyte.blogspot.com". The right-most label is called top level domain. In the above example .com is the top level domain. The other labels, which are to the left of top level domain, are called subdomains or subdivisions of the top level domain. The subdivisions may be up to 127. Any label may not start or end with a hyphen.

  2. Conversion of Domain or URL into IP address

    3. Through DNS, the domain, web address or URL you have entered is converted into its IP address. For example, google.com has the IP address: "173.194.124.39". You can find out the IP address of almost any website by pinging. Just open command-prompt, or cmd.exe. And then use ping command, such as: 

    ping rusted.blogspot.com
    You can use the IP address to open a website by just entering the IP address into the address bar.

  3. IP address gets transferred to server

    4. The IP address gets transferred to a server, usually ISP's server. The server looks up for the IP address in its memory, if found then it will redirect the client to the server which is hosting the requested webpage. If not found, then it will transfer the requested address to another server. The cycle will continue till the IP address of the server which hosts the requested webpage is found. Once found, that address will be return to the client who requested it in the first place.

  4. Caching of the webpage server's address

    5. If the server which gets the client's request already has the requested webpage's hosting server's address, then it will just redirect the client to the requested webpage. If not, the above step will take place.

    After getting the IP address from another server, the ISP's server will cache the address or store it in its memory, in case there are more requests for the same webpage.

  5. Back to the client

    6. The client gets redirected to the requested webpage.
